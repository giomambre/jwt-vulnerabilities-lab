Task 2 — RS256 → HS256 Algorithm Confusion

The server is supposed to use RS256 but also accepts HS256. The RSA public key is publicly exposed at /task2/public.pem. An attacker re-signs a tampered token using the public key as the HMAC-SHA256 secret — the server verifies it successfully.

Task 3 — kid Header Path Traversal

Task 4 — jwk Header Injection

The server reads the jwk field from the JWT header and uses whatever public key it finds there. An attacker generates their own RSA key pair, embeds the public key in the header, signs with the private key, and the server verifies it — using the attacker's own key.
